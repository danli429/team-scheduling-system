# 团队排班系统

一个功能完善的团队活动排班管理系统，用于管理定期分享、团建等团队活动的人员安排。

## 功能特点

### 核心功能
- **成员管理**：添加、编辑、删除团队成员，支持活跃/暂停状态切换
- **活动管理**：创建不同类型的团队活动，设置活动周期（按天/周/月）
- **自动排班**：提供三种排班算法
  - 轮流排班：按顺序轮流分配
  - 随机分配：随机选择成员
  - 均衡分配：优先分配给参与次数较少的成员
- **日历展示**：直观的日历视图显示排班情况
- **通知提醒**：可设置提前N天通知即将负责活动的成员
- **数据管理**：支持数据导出/导入，方便备份和迁移

### 技术特点
- 纯前端实现，无需后端服务器
- 数据存储在浏览器本地（LocalStorage）
- 响应式设计，支持PC和移动端访问
- 现代化界面，操作简单直观

## 快速开始

### 1. 打开系统
直接用浏览器打开 `index.html` 文件即可使用。

### 2. 添加成员
1. 点击顶部导航的"成员管理"标签
2. 点击"添加成员"按钮
3. 填写成员姓名和邮箱
4. 选择成员状态（活跃/暂停）
5. 点击"添加"保存

### 3. 创建活动
1. 点击"活动管理"标签
2. 点击"添加活动"按钮
3. 输入活动名称（如：技术分享会、团队建设）
4. 填写活动描述
5. 设置活动频率（如：每2周一次）
6. 点击"添加"保存

### 4. 生成排班
1. 回到"排班日历"标签
2. 点击"生成排班"按钮
3. 选择排班的开始和结束日期
4. 点击"生成"，系统将自动分配所有活动的负责人

### 5. 查看排班
- 在日历中，有活动的日期会显示蓝色背景
- 点击具体日期可查看当天的活动详情
- 在"即将到来的活动"区域查看近期排班

## 使用说明

### 成员管理
- **添加成员**：点击"添加成员"按钮，填写姓名和邮箱
- **编辑成员**：在成员列表中点击"编辑"按钮修改信息
- **删除成员**：点击"删除"按钮移除成员
- **状态管理**：
  - 活跃：参与排班分配
  - 暂停：不参与排班（如休假、离职等）

### 活动管理
- **活动类型**：可创建多种活动类型，如：
  - 技术分享会
  - 团队建设
  - 项目复盘
  - 知识培训
- **活动频率**：支持按天、周、月设置周期
  - 例如：每2周一次技术分享
  - 例如：每1个月一次团建活动

### 排班算法
在"系统设置"中可选择不同的排班算法：

1. **轮流排班**（推荐）
   - 按顺序轮流分配，确保每个人机会均等
   - 适合大多数场景

2. **随机分配**
   - 随机选择负责人
   - 适合不需要固定顺序的场景

3. **均衡分配**
   - 优先分配给参与次数少的成员
   - 适合需要严格平衡工作量的场景

### 通知提醒
1. 在"系统设置"中启用"通知提醒"
2. 设置提前提醒天数（默认3天）
3. 浏览器会在指定时间发送通知
4. 首次使用需要授权浏览器通知权限

### 数据管理

#### 导出数据
1. 进入"系统设置"标签
2. 点击"导出数据"按钮
3. 系统会下载一个JSON格式的备份文件
4. 建议定期备份数据

#### 导入数据
1. 点击"导入数据"按钮
2. 选择之前导出的JSON文件
3. 数据将被恢复到系统中

#### 清除数据
- 点击"清除所有数据"可重置系统
- 此操作不可恢复，请谨慎使用

## 使用场景示例

### 场景1：技术分享会排班
1. 添加团队10名成员
2. 创建活动"技术分享会"，频率"每1周"
3. 生成未来3个月的排班
4. 系统自动分配，确保每人每10周轮值一次

### 场景2：多活动并行管理
1. 创建"技术分享会"（每周一次）
2. 创建"团队建设"（每月一次）
3. 创建"项目复盘"（每两周一次）
4. 一次生成所有活动的排班计划

### 场景3：成员状态变更
- 成员请假：将状态改为"暂停"，不影响已有排班
- 成员回归：改回"活跃"状态，重新生成排班即可参与

## 注意事项

### 数据安全
- 所有数据存储在浏览器本地
- 清除浏览器缓存会导致数据丢失
- 建议定期导出数据备份

### 浏览器兼容性
- 推荐使用现代浏览器：
  - Chrome 90+
  - Firefox 88+
  - Safari 14+
  - Edge 90+

### 最佳实践
1. **定期备份**：每月导出一次数据
2. **合理设置周期**：根据团队规模设置活动频率
3. **及时更新状态**：成员变动时及时更新状态
4. **重新生成排班**：团队成员或活动变更后，重新生成排班计划

## 常见问题

### Q: 如何修改已生成的排班？
A: 目前需要重新生成排班。建议在生成前确认好成员和活动信息。

### Q: 可以为某个活动指定特定负责人吗？
A: 当前版本采用自动分配，后续可通过手动调整实现。

### Q: 数据能否在不同电脑间同步？
A: 当前数据存储在本地，可通过导出/导入功能在不同设备间迁移数据。

### Q: 通知提醒不工作？
A: 请检查：
1. 是否授予了浏览器通知权限
2. 系统设置中是否启用了通知
3. 浏览器是否支持通知API

### Q: 可以管理多个团队吗？
A: 当前版本只支持一个团队，可通过导出/导入切换不同团队的数据。

## 技术架构

### 文件结构
```
scheduling-system/
├── index.html      # 主页面
├── styles.css      # 样式文件
├── app.js          # 核心逻辑
└── README.md       # 说明文档
```

### 核心类
- **DataManager**：数据管理，负责CRUD操作和本地存储
- **ScheduleGenerator**：排班算法，实现三种分配策略
- **NotificationManager**：通知管理，处理提醒功能
- **UIManager**：界面管理，处理所有用户交互

### 数据结构
```javascript
// 成员
{
  id: string,
  name: string,
  email: string,
  status: 'active' | 'inactive',
  participationCount: number
}

// 活动
{
  id: string,
  name: string,
  description: string,
  frequency: number,
  frequencyUnit: 'days' | 'weeks' | 'months'
}

// 排班
{
  id: string,
  activityId: string,
  activityName: string,
  memberId: string,
  memberName: string,
  date: string,
  notified: boolean
}
```
